    • Overview

      This project followed 6 weeks worth of lab content for the Instruction Architectures and Compilers module in 2nd year. This consisted of 3 2-week labs focused on the core components of a compiler:

      • Weeks 1-2 assessed our ability to work with a lexer using Flex to break down input streams of text into words, decimals and fractions then calculate the sum of the numbers in the input text and form a histogram from the words detected.
      • Weeks 3-4 were centred on parsing using Yacc. Input mathematical expressions would be passed through a pre-written lexer and then through our parser. Forming an AST (Abstract Syntax Tree) using inheritance in C++, the parsed input would have it's type and variables identified. This then allowed us to call a function which would compare the result of the parsed expression (where values for the variables have been given) with a given test-case answer to verify if the expression was parsed correctly.
        Our job was to write the remaining parser logic and fill in the relevant functions in the AST.
      • The final weeks (5-6) were finally focused on compiling parsed input into machine code. The target language was something made up by the professor for the purpose of this task only.

      After the labs, we commenced the project of writing a whole compiler from the ground up. This began with writing a lexer using Flex to break down input streams of text into its tokens. Afterwards, a parser would attempt to identify constructs such as if statements, while loops, variable assignments etc. We created an AST using a series of header files written in C++ to then handle compilation of the parsed information.

      As well as using Yacc, Flex and C++, I learnt how Makefiles work to alleviate compilation of our files.
